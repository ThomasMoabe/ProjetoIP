Projeto

Especificação de Requisitos do Projeto

Restrições:


  • No mínimo três classes básicas (Ex. Conta, Livro, Funcionário, DVD);

    • Essas devem ser validadas juntamente com o tema do projeto por um monitor.

  • Deve haver pelo menos uma relação entre as classes básicas;

  • Três classes coleção de dados para cada classe básica (Ex. No caso da classe básica Conta, seriam os seguintes repositórios RepositorioContasArray, RepositorioContasLista, RepositorioContasArquivo);

  • Uma classe coleção de negócio que usa as coleções de dados para armazenar os objetos das classes básicas e, se for o caso, define regras (validações) para cadastrar, remover, atualizar e procurar objetos no sistema. (Ex. CadastroContas);


  • Deve ser criada uma classe fachada que deve ter como atributos todos os cadastros do sistema. Por exemplo, no caso de um projeto com três classes básicas devem haver nove coleções de dados (serão vistos logo mais), três classes coleção de negócio e uma classe fachada com 3 atributos (os 3 cadastros). A classe fachada é quem deve fazer validações que envolvem mais de um cadastro;

  • Usem pacotes;

  • Usem herança (onde for necessário e viável);

  • Deve haver pelo menos um exemplo de classe abstrata (já serve como exemplo de herança);

  • Defina interfaces negócio-dados para todas as coleções de dados e para os iterators (como mostrado em sala na aula de interfaces);

  • Definam uma Classe Programa no pacote gui que testa a classe fachada (estes testes devem ser fixos, apenas para ver se os métodos funcionam e servem para vocês testarem o projeto antes de implementar a interface gráfica);

  • Usem exceções como mostrado em sala;

  • Implementem uma interface gráfica.




Devem ser criados três repositórios:
  • Array
  • TAD
    • Deve ser escolhida uma das estruturas de dados vistas em sala.
  • Arquivo
    • Os dados devem ser persistentes, ou seja, quando o programa executar novamente utilizando este repositório, deverá acessar as informações já armazenadas
    • Os dados devem ser armazenados numa planilha[Excel]. Pra isso, recomenda-se o uso da biblioteca Jakarta POI.

A escolha do repositório deve ser feita ao iniciar o programa através da leitura de um arquivo chamado config.txt.


Interfaces
  • Repositórios
    • Inserir
    • Procurar
    • Atualizar
    • Remover

  • Iterator
    • next
    • hasNext
    • getIterator

GUI
  • Deve ser criada uma interface gráfica, contendo todas as funcionalidades do projeto.




Relatório
  • Deve ser possível gerar um relatório com informações gerais do sistema (Ex. faturamento diário em um mês);
  • O resultado deve ser ordenado por algum parâmetro (Ex. faturamento, data);
  • O relatório deve ser mostrado em tela;
  • Deve ser salvo em um documento txt com nome e diretório escolhidos pelo usuário;
  • Cada relatório deve implementar a interface Iterable e ter o método iterator para navegar sobre os itens do relatório, escrevendo progressivamente as strings no txt;
  • Além disso todos os relatórios devem ser armazenados em uma coleção e serializados no arquivo relatorios.ip;
  • Os relatórios já emitidos podem ser acessados pelo programa através de relatorios.ip.
